# فین‌ترک — یادداشت‌های پیاده‌سازی و راهنمای تبدیل به APK

این سند شامل این موارد است: ساختار نهایی فایل‌ها، نحوه اجرای برنامه به‌صورت لوکال، تصمیمات مهم پیاده‌سازی، و راهنمای گام‌به‌گام تبدیل اپلیکیشن وب به یک فایل APK اندروید با استفاده از **Capacitor** (پیشنهادی) یا **Cordova**.

---

fintrack-app/

## ۲. اجرای برنامه به‌صورت لوکال

از آنجایی که برنامه از Service Worker استفاده می‌کند، باید از طریق **http(s)** اجرا بشه، نه با باز کردن مستقیم فایل (مسیر `file://`) — چون Service Worker روی `file://` کار نمی‌کنه.

ساده‌ترین راه‌ها:

**روش الف — اکستنشن Live Server در VS Code**
۱. پوشه `fintrack-app` را در VS Code باز کن.
۲. اکستنشن «Live Server» را نصب کن.
۳. روی `index.html` کلیک راست کن ← «Open with Live Server».

**روش ب — سرور داخلی پایتون**
```bash
cd fintrack-app
python3 -m http.server 8080
```
سپس در مرورگر آدرس `http://localhost:8080` را باز کن.

**روش ج — ابزار http-server در Node**
```bash
npx http-server fintrack-app -p 8080
```

## ۳. تصمیمات کلیدی پیاده‌سازی (چرا این‌طور ساخته شد)

- **بدون فریم‌ورک**: همه‌چیز با جاوااسکریپت خالص و الگوی ماژولار نوشته شده تا کد از بالا به پایین راحت خوانده بشه، به‌راحتی قابل بررسی توسط استاد باشه، و هیچ مرحله build‌ای نیاز نداشته باشه.
- **LocalStorage به‌عنوان تنها لایه داده**: تمام عملیات CRUD از طریق `StorageService` انجام می‌شه، پس اگر بعداً بخوای LocalStorage را با IndexedDB یا یک بک‌اند واقعی جایگزین کنی، فقط کافیه یک فایل رو تغییر بدی.
- **هش کردن رمز عبور**: تابع `simpleHash()` در `script.js` یک هش سبک و غیررمزنگاری‌شده است که فقط به این دلیل استفاده شده که این برنامه کاملاً آفلاین است و سروری در کار نیست. این روش **برای محصول واقعی مناسب نیست** — اگر در دفاع از پروژه پرسیده شد، به‌وضوح توضیح بده که یک محصول واقعی باید رمز عبور را در سمت سرور با الگوریتم مناسب (bcrypt، Argon2) هش و بررسی کنه.
- **چیدمان موبایل-فرست با قاب گوشی**: برنامه با حداکثر عرض `480px` وسط‌چین شده، بنابراین همیشه شبیه یک اپ واقعی گوشی به نظر می‌رسه — همین موضوع باعث میشه اسکرین‌شات‌هایی که برای گزارش پروژه می‌گیری هم حرفه‌ای‌تر به نظر برسن.
- **ذخیره خودکار (Auto-save)**: هر عملیات افزودن/ویرایش/حذف بلافاصله در LocalStorage نوشته می‌شه — نیازی به دکمه «ذخیره» جداگانه نیست که فراموشش کنی. این‌طوری قابلیت auto-save به‌طور طبیعی برآورده می‌شه.
- **Chart.js با نسخه پشتیبان**: تگ `<head>` در `index.html` اول از CDN تلاش می‌کنه، و اگر CDN در دسترس نبود، خودکار به فایل لوکال سوییچ می‌کنه. این موضوع وقتی برنامه رو به APK تبدیل می‌کنی خیلی مهمه (پایین‌تر توضیح داده شده).
- **فارسی‌سازی کامل و راست‌به‌چپ**: کل رابط کاربری فارسی است و جهت صفحه (`dir="rtl"`) روی حالت راست‌به‌چپ تنظیم شده. تاریخ‌ها با `toLocaleDateString('fa-IR')` به‌صورت شمسی نمایش داده می‌شن (خود داده در پس‌زمینه به‌صورت میلادی ذخیره می‌مونه تا با ورودی‌های `<input type="date">` سازگار باشه). واحد پول پیش‌فرض «تومان» است و از تنظیمات قابل تغییره.

## ۴. آماده‌سازی Chart.js برای استفاده آفلاین/APK

وقتی برنامه در قالب Capacitor/Cordova بسته‌بندی و به‌صورت APK نصب می‌شه، معمولاً هنگام دمو کردن به‌صورت آفلاین (مثلاً توی کلاس بدون وای‌فای) دسترسی زنده به اینترنت از دست می‌ره. در این حالت تگ CDN اسکریپت کار نمی‌کنه و نمودارها نمایش داده نمی‌شن — پس بهتره یک نسخه لوکال همراه پروژه داشته باشی.

**مراحل:**
۱. روی یک کامپیوتر که به اینترنت وصله، این فایل رو دانلود کن:
   `https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js`
۲. آن را با نام `fintrack-app/lib/chart.min.js` ذخیره کن.
۳. همین کافیه — کد fallback که از قبل در `index.html` قرار داره، خودکار در صورت شکست CDN از همین فایل لوکال استفاده می‌کنه.
۴. همچنین خط مربوط به `lib/chart.min.js` را در آرایه `APP_SHELL_FILES` داخل `service-worker.js` از حالت کامنت خارج کن تا برای استفاده کاملاً آفلاین کش بشه.

## ۵. تبدیل به APK اندروید با Capacitor (پیشنهادی)

Capacitor (ساخته تیم Ionic) روشی مدرن و به‌روز برای بسته‌بندی یک اپ وب داخل یک پوسته بومی اندروید/iOS است. این کار نیازی به هیچ تغییری در HTML/CSS/JS شما فراتر از چیزی که در این پروژه انجام شده ندارد.

### پیش‌نیازها
- نصب [Node.js](https://nodejs.org/) (نسخه LTS)
- نصب [Android Studio](https://developer.android.com/studio) (برای Android SDK + شبیه‌ساز و ابزارهای build)

### گام‌به‌گام

**۱. راه‌اندازی یک پروژه Node در پوشه برنامه**
```bash
cd fintrack-app
npm init -y
```

**۲. نصب Capacitor**
```bash
npm install @capacitor/core
npm install -D @capacitor/cli
```

**۳. مقداردهی اولیه Capacitor**
```bash
npx cap init "فین‌ترک" "com.yourname.fintrack" --web-dir="."
```
- به‌جای `com.yourname.fintrack` یک شناسه اختصاصی خودتان (به سبک دامنه معکوس) بگذارید.
- `--web-dir="."` به Capacitor می‌گوید فایل‌های وب شما (`index.html`, `style.css`, `script.js` و غیره) در همین پوشه فعلی هستند — چون این یک پروژه ساده HTML/CSS/JS است و نیازی به مرحله build جداگانه نیست.

**۴. اضافه کردن پلتفرم اندروید**
```bash
npm install @capacitor/android
npx cap add android
```
این دستور یک پوشه `android/` می‌سازد که شامل یک پروژه کامل Android Studio است که اپ وب شما را داخل یک WebView بارگذاری می‌کند.

**۵. کپی کردن فایل‌های وب به پروژه بومی**
هر بار که HTML/CSS/JS رو تغییر دادی، این دستور را اجرا کن:
```bash
npx cap copy android
```

**۶. باز کردن پروژه در Android Studio**
```bash
npx cap open android
```
Android Studio پروژه بومی را باز می‌کند. از آنجا می‌توانید:
- روی یک شبیه‌ساز یا گوشی متصل اجرا کنید (دکمه ▶ Run)
- یک APK نهایی امضاشده بسازید از مسیر **Build → Generate Signed Bundle / APK**

**۷. ساخت سریع یک APK آزمایشی (بدون نیاز به رابط Android Studio)**
```bash
cd android
./gradlew assembleDebug
```
فایل APK در این مسیر ساخته می‌شود:
`android/app/build/outputs/apk/debug/app-debug.apk`

می‌تونی این فایل `.apk` را مستقیم روی هر گوشی اندرویدی منتقل کنی و نصبش کنی (ابتدا گزینه «نصب از منابع ناشناس» را در گوشی فعال کن).

### نکات مخصوص این پروژه
- LocalStorage دقیقاً همانند مرورگر، داخل WebView مربوط به Capacitor هم کار می‌کند — نیازی به تغییر کد نیست.
- Service Worker (`service-worker.js`) روی نسخه‌های مدرن اندروید داخل WebView کپسیتور هم کار می‌کند و کش آفلاین را حتی داخل اپ بومی هم فراهم می‌کند.
- چون همه‌چیز با چیدمان قاب گوشی (`max-width: 480px`) وسط‌چین شده، برنامه روی گوشی‌های واقعی اندروید بدون نیاز به تنظیم اضافه CSS، درست و بومی به نظر می‌رسد.
- اگر آیکون برنامه را عوض کردی، فایل‌های `assets/icons/icon-192.png` و `icon-512.png` را دوباره بساز و آیکون‌های launcher بومی زیر `android/app/src/main/res/mipmap-*/` را هم به‌روزرسانی کن (ابزار `@capacitor/assets` می‌تونه این کار رو خودکار از روی همین دو فایل PNG انجام بده).

## ۶. روش جایگزین: تبدیل با Cordova

Cordova روش قدیمی‌تر و همچنان پشتیبانی‌شده به‌جای Capacitor است. مراحل از نظر مفهومی مشابه است:

```bash
npm install -g cordova
cordova create fintrack-cordova com.yourname.fintrack FinTrack
```
سپس فایل‌های `index.html`، `style.css`، `script.js`، `manifest.json`، `assets/` و `lib/` را داخل پوشه `www/` پروژه ساخته‌شده کپی کن (جایگزین فایل‌های پیش‌فرض آن). سپس:
```bash
cd fintrack-cordova
cordova platform add android
cordova build android
```
فایل APK در مسیر `platforms/android/app/build/outputs/apk/` تولید می‌شود.

> **پیشنهاد:** به‌جای Cordova از Capacitor استفاده کن — به‌طور فعال نگهداری می‌شه، پشتیبانی بهتری از اندروید/iOS مدرن داره، و ابزارهایش (`npx cap open android`) برای مبتدی‌ها راحت‌تره.

## ۷. چک‌لیست پیش از تحویل پروژه

- [ ] کل مسیر کاربر رو تست کن: ثبت‌نام ← ورود ← افزودن درآمد/هزینه ← تنظیم بودجه ← مشاهده نمودارها ← افزودن/ویرایش/حذف مخاطب ← تغییر رمز عبور ← خروج ← ورود دوباره.
- [ ] روی یک صفحه واقعی موبایل باریک (یا حالت Device Toolbar در Chrome DevTools) تست کن که چیدمان درست نمایش داده بشه.
- [ ] حالت تاریک رو فعال کن و همه صفحات رو از نظر کنتراست/خوانایی دوباره چک کن.
- [ ] مطمئن شو LocalStorage بعد از رفرش کردن تب مرورگر باقی می‌مونه.
- [ ] قبل از بسته‌بندی APK برای دموی آفلاین، نسخه لوکال Chart.js رو دانلود کن (بخش ۴).
- [ ] اگر پروژه رو به استاد ارائه می‌دی، به‌وضوح بگو که هش رمز عبور فقط سمت کلاینت و مناسب یک پروژه آموزشی آفلاینه، نه برای محصول واقعی.

## ۸. پیشنهادهای توسعه آینده (نکات جایزه برای گزارش پروژه)

- پشتیبانی چندکاربره (در حال حاضر برنامه فقط یک حساب کاربری لوکال روی هر دستگاه/مرورگر پشتیبانی می‌کنه).
- خروجی گرفتن CSV از تراکنش‌ها (علاوه بر خروجی JSON فعلی).
- تراکنش‌های تکرارشونده (مثلاً افزودن خودکار اجاره ماهانه).
- اعلان‌های هشدار بودجه (نیاز به Notifications API + متد `showNotification` در Service Worker داره).
- همگام‌سازی و پشتیبان‌گیری ابری با یک بک‌اند واقعی (Firebase، Supabase، یا یک REST API اختصاصی) اگر محدودیت آفلاین‌بودن در آینده برداشته بشه.
